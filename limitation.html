Limitations

Limitations:

  1. The maximum number of OPTIGA session usage is limited to 4 for the optiga crypt instances.
  2. Since OPTIGA is a singleton resource, the number of instances that can run in parallel is limited to 6 (1 active instance and 5 instance will be queued up internally). To increase the number of parallel instances re-configure the macro OPTIGA_CMD_MAX_REGISTRATIONS in optiga_lib_config.h
  3. Third-party libraries such as mbedtls might invoke memory allocation functions during command/response protection(shielded connection) operations.
    There could be collision during memory allocation, if a create API from service layer is invoked at the same time.
  4. As the RSA key generation can go beyond 50 seconds, the default timeout of ifx i2c protocol (TL_MAX_EXIT_TIMEOUT) is set to 180 seconds. Otherwise, 10 seconds is sufficient.
  5. Disable of OPTIGA_CRYPT_RANDOM_ENABLED feature macro must not be done, if OPTIGA COMMS shielded connection feature macro is enabled in optiga_lib_config.h
  6. When all cryptographic feature macros and OPTIGA COMMS shielded connection feature macro is disabled in optiga_lib_config.h,
    then optiga_crypt.c will throw a compilation warning of function "optiga_crypt_reset_protection_level" was declared but never referenced.
